# Decision Log

## 2026-05-20: PC 데스크톱 우선

결정:
- SermonScript는 Windows PC 데스크톱 앱으로 먼저 개발합니다.

근거:
- 긴 설교 녹음 처리에는 PC의 CPU/GPU, 메모리, 파일 관리, 편집 UI가 유리합니다.
- 모바일은 녹음/확인/공유 보조 앱으로 나중에 검토합니다.

## 2026-05-20: Python + PySide6 채택

결정:
- Python 3.11+, PySide6, faster-whisper, FFmpeg, SQLite 조합을 사용합니다.

근거:
- STT, 오디오, 후처리 생태계가 Python 중심입니다.
- CLI와 GUI를 같은 core 모듈로 재사용하기 쉽습니다.

## 2026-05-20: YouTube URL 입력 후순위

결정:
- YouTube URL 입력은 v1.0에서 제외하고 v1.5 또는 v2.0 후보로 둡니다.

근거:
- 저작권/이용 권한 리스크가 있습니다.
- 핵심 로컬 파일 STT 파이프라인 안정화가 먼저입니다.
- 나중에 input source 계층으로 분리 구현하면 됩니다.

## 2026-05-20: GUI 프레임워크 Tauri + React 하이브리드 대전환

결정:
- 데스크톱 GUI 환경을 기존 PySide6(Python)에서 **Tauri 2.0 (Rust) + React (TypeScript + Tailwind CSS + shadcn/ui)** 하이브리드 아키텍처로 대전환합니다.
- 기존의 견고한 Python 핵심 엔진(전처리, STT, 원문 대조, SQLite 스키마 v2 등)은 PyInstaller를 통해 GUI 모듈이 완전히 제거된 경량 사이드카(`sermonscript-sidecar`)로 패키징하여 백그라운드 프로세스로 통합합니다.

근거:
- **시각적 한계 극복**: PySide6의 기본 스타일링은 상용 등급의 유려한 최신 데스크톱 UI/UX 및 다크모드, 글래스모피즘 등을 유연하게 처리하기 어렵습니다. React + Tailwind CSS 및 shadcn/ui 생태계를 활용하여 극상의 프리미엄 aesthetics를 구현합니다.
- **풍부한 미디어 인터랙션**: 향후 오디오 파형(Waveform) 렌더링, 오디오 재생 연동 및 세그먼트별 하이라이팅 등 복잡한 미디어 인터랙션을 다루는 데는 크롬 기반 웹 환경(Tauri WebView)이 훨씬 강력하고 이식성이 높습니다.
- **안정적인 프로세스 격리**: 무거운 AI 연산(faster-whisper)과 핵심 비즈니스 로직을 백그라운드 Python 사이드카 프로세스에 온전히 위임하고 UI 스레드와 완벽히 격리함으로써, 앱의 크래시와 프리징 현상을 차단하고 탁월한 프론트엔드 반응성을 얻을 수 있습니다.

## 2026-05-20: Tauri 하이브리드 E2E 사이드카 연동 및 프리미엄 3D 앱 아이콘 탑재 성공

결정:
- `--json` 스펙을 적용하여 새로 컴파일된 파이썬 사이드카 바이너리 패키징 및 Tauri binaries 내부 배치를 성공적으로 완료합니다.
- SermonScript의 아이덴티티를 투영한 초프리미엄 3D 디자인의 앱 아이콘을 개발하여 데스크톱 브랜드 자산으로 공식 탑재합니다.
- Rust 비동기 IPC 링킹과 React 다크테마 글래스모피즘 화면을 연동하여 E2E 수동 검증 및 90개 단위 테스트, Ruff 린트, doctor 자가진단 100% 무결성을 확보합니다.

근거:
- **실전 검증 무결성**: 90개의 파이썬 테스트 스위트 전원이 PASS되었고, Ruff 자동 수정을 통해 고품질의 코드베이스 스타일을 확립했습니다.
- **향상된 브랜드 아이덴티티**: 밋밋한 기본 로고 대신 3D 소리 파형과 성경 두루마리가 가미된 프리미엄 아이콘을 탑재하여 상용화 단계의 완성도를 대폭 끌어올렸습니다.
