# Deferred Feature — YouTube URL Import (v1.0 제외)

## 1. 상태

YouTube URL 입력은 v1.0 범위에서 **제외**합니다. 현재 GUI/CLI는
이 기능을 노출하지 않으며, 의도적으로 코드 경로도 만들지 않습니다.

후보 버전:

```text
v1.5 또는 v2.0
```

이 문서는 단일 진실 공급원(SSOT)입니다. `docs/youtube-import-deferred.md`
(Goal 1 시점에 작성된 동일 주제 문서)와 내용이 충돌하는 경우 이 문서의
정책을 따릅니다.

## 2. 이유

- 저작권 및 이용 권한 문제가 발생할 수 있습니다.
- 온라인 다운로드 기능은 법적·정책적 리스크가 있습니다.
- 핵심 제품인 로컬 오디오 STT 파이프라인 안정화가 먼저입니다.
- 입력 소스 기능은 별도 모듈로 붙이는 것이 안전합니다.

## 3. 나중에 구현할 경우의 처리 흐름

```text
YouTube URL 입력
→ URL 검증
→ 저작권/이용 권한 경고 표시
→ 사용자가 명시적으로 확인
→ 오디오 트랙 추출 또는 가져오기 (외부 도구)
→ 로컬 임시 오디오 파일 생성
→ 기존 오디오 전처리 파이프라인에 전달
→ STT 변환
→ 편집/Export
```

## 4. 필수 경고 문구

YouTube/온라인 입력이 활성화될 경우 다음 문구를 GUI/CLI 모두에서 명시적으로
표시해야 합니다.

```text
이 기능은 본인이 권리를 보유했거나 변환 권한이 있는 콘텐츠에만 사용해야 합니다.
PulpitInk는 저작권 침해 목적의 다운로드나 변환을 권장하지 않습니다.
사용자는 콘텐츠 처리 권한을 직접 확인할 책임이 있습니다.
```

## 5. 설계 원칙

YouTube 기능은 STT core에 절대 들어가면 안 됩니다.

```text
input_sources/youtube_url.py
→ local temp audio file
→ existing audio pipeline (pulpit_ink.core.audio)
```

## 6. v1.0 공통 규칙

```text
YouTube URL 입력, 온라인 다운로드, 클라우드 기능은 v1.0 범위에서 제외한다.
현재는 로컬 오디오/비디오 파일 입력만 지원한다.
```

## 7. v1.0 작업자 가이드

- GUI 코드에 URL 입력 위젯을 추가하지 않는다.
- CLI에 `--url` 또는 `--youtube` 옵션을 추가하지 않는다.
- 이 문서가 갱신되기 전까지 어떤 형태로도 활성화하지 않는다.
