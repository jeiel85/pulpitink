# YouTube URL Import Policy

## 1. 상태

이 문서는 과거의 “v1.0 제외” 결정을 보존하되, 현재 구현 상태에 맞춰 갱신한
YouTube URL 입력 정책입니다.

현재 상태:

```text
v0.4.4 이후 GUI에서 저작권/이용 권한 고지 동의 후 YouTube URL 입력 지원
v0.4.5 이후 yt-dlp 설치 상태 진단 및 원클릭 설치 UI 지원
```

이 문서는 단일 진실 공급원(SSOT)입니다. `docs/youtube-import-deferred.md`
(Goal 1 시점에 작성된 동일 주제 문서)는 과거 결정 기록으로만 취급합니다.

## 2. 이유

- 저작권 및 이용 권한 문제가 발생할 수 있습니다.
- 온라인 다운로드 기능은 법적·정책적 리스크가 있습니다.
- 핵심 STT 파이프라인은 계속 로컬 파일 입력을 기준으로 유지해야 합니다.
- 입력 소스 기능은 STT core와 분리해야 합니다.

## 3. 현재 처리 흐름

```text
YouTube URL 입력
→ URL 검증
→ 저작권/이용 권한 경고 표시
→ 사용자가 명시적으로 확인
→ yt-dlp 설치 상태 진단 또는 사용자 동의 기반 설치
→ 오디오 트랙 추출 또는 가져오기
→ 로컬 임시 오디오 파일 생성
→ 기존 오디오 전처리 파이프라인에 전달
→ STT 변환
→ 편집/Export
```

## 4. 필수 경고 문구

YouTube 입력 사용 전 다음 취지의 문구를 GUI/문서에 명시합니다.

```text
이 기능은 본인이 권리를 보유했거나 변환 권한이 있는 콘텐츠에만 사용해야 합니다.
PulpitInk는 저작권 침해 목적의 다운로드나 변환을 권장하지 않습니다.
사용자는 콘텐츠 처리 권한을 직접 확인할 책임이 있습니다.
```

## 5. 설계 원칙

YouTube 기능은 STT core에 절대 들어가면 안 됩니다.

```text
input source / youtube downloader
→ local temp audio file
→ existing audio pipeline (pulpit_ink.core.audio)
```

## 6. v1.0 공통 규칙

```text
YouTube URL 입력은 고지 동의 기반 opt-in 입력 소스로만 제공한다.
일반 온라인 다운로드, 클라우드 기능, DRM/접근 제한 우회는 v1.0 범위에서 제외한다.
```

## 7. v1.0 작업자 가이드

- 기존 YouTube 입력 기능을 확장할 때는 권리 고지, 사용자 확인, 안전한 임시 파일 처리, 로그 개인정보 보호를 먼저 확인합니다.
- 다운로드 우회 기능, 로그인/쿠키 처리, 유료 서비스 연동, 클라우드 업로드 기능은 구현하지 않습니다.
- STT core는 로컬 파일 입력만 받도록 유지하고, YouTube 처리는 로컬 임시 오디오를 만드는 입력 소스 계층에 둡니다.
