# 알려진 제한사항 (v1.0)

PulpitInk v1.0 에서 의도적으로 제외했거나 현재 한계로 남아 있는 항목을 한곳에 정리한 문서입니다.
릴리즈 노트와 README, 사용자 가이드의 동일 주제 절은 모두 이 문서를 출처로 합니다.

## 1. 입력 소스

- **로컬 오디오/비디오 파일만 지원합니다.** 지원 확장자: `mp3`, `wav`, `m4a`, `aac`, `flac`, `ogg`, `mp4`.
- **YouTube URL 입력 / 온라인 다운로드 / 스트리밍 입력은 포함되지 않습니다.** 설계와 사유는 [deferred-youtube-import.md](deferred-youtube-import.md) 참고. v1.5 또는 v2.0 대상.
- **마이크 실시간 입력은 미지원** 입니다. 녹음은 외부 도구로 진행해 파일로 저장한 뒤 변환하세요.

## 2. 처리 / 모델

- **화자 분리(diarization) 는 제공하지 않습니다.** 결과의 `speaker` 컬럼은 항상 비어 있습니다.
- **STT 모델 파일은 패키지에 번들되지 않습니다.** 최초 사용 시 자동 다운로드되며 사용자 데이터 디렉터리에 캐시됩니다. 오프라인 환경에서는 모델 캐시를 미리 복사해야 합니다.
- **CUDA 지원은 사용자 환경 의존입니다.** GPU 가속을 사용하려면 호환되는 NVIDIA 드라이버와 CUDA / cuDNN 이 사전 설치되어 있어야 합니다.
- **언어 자동 감지를 항상 신뢰하지 마세요.** 짧은 오디오나 잡음이 많은 녹음에서는 `--language ko` 처럼 명시적으로 지정하는 것을 권장합니다.

## 3. 외부 의존성

- **FFmpeg 는 번들되지 않습니다.** `ffmpeg` / `ffprobe` 가 PATH 에서 호출 가능해야 합니다. 설치 안내는 [user-guide.md §1.1](user-guide.md#11-사전-준비) 참고.
- **PySide6 (LGPL-3.0) 정책.** GUI 옵션 의존성으로만 분리되어 있으며, 재배포 시 LGPL 조건을 따라야 합니다. [THIRD_PARTY_NOTICES.md](../THIRD_PARTY_NOTICES.md) 참고.

## 4. 패키징 / 플랫폼

- **공식 패키징은 Windows Portable ZIP 만 제공합니다.** macOS·Linux 는 소스 설치(`pip install -e .`) 로 사용 가능하지만 자동화된 빌드 / 회귀 검증 대상은 아닙니다.
- **MSI / 인증서 서명 설치 관리자는 아직 제공하지 않습니다.** Portable ZIP 만 v1.0 산출물입니다.
- **앱 아이콘은 선택 사항** 입니다. 릴리즈 직전에 추가될 수 있습니다.

## 5. 데이터 / 동작 정책

- **`raw_text` 는 절대 덮어쓰지 않습니다.** 사용자의 편집은 `edited_text` 에, 후처리 결과는 `clean_text` 에만 저장됩니다.
- **자동 교정은 보수적입니다.** 일반 문장은 자동 교체하지 않으며, 성경 구절·고유명사·사용자 사전 기반 후보만 `pending` 으로 남기고 사용자의 검토를 거칩니다.
- **원본 오디오 파일은 절대 수정되지 않습니다.** 모든 전처리는 `cache/jobs/<job_id>/processed.wav` 에 새 파일로 작성됩니다.

## 6. 보안 / 프라이버시

- **로컬-우선 설계입니다.** 오디오/텍스트가 외부로 전송되지 않습니다. 클라우드 / 외부 API 송신은 v1.0 범위에서 제외입니다.
- **로그에는 전체 transcript 가 남지 않습니다.** 진단용 메타데이터만 기록합니다. 자세한 로그가 필요한 경우 `--verbose` 옵션을 사용하세요.
- **최근 파일 기록 비활성화** 옵션은 v1.x 후속 작업입니다 (release-checklist 추적).

## 7. UI / 편집기

- **편집기는 PySide6 가 설치된 경우에만 GUI 에서 열 수 있습니다.** CLI 만 사용한다면 `PulpitInk corrections` 와 `PulpitInk jobs export` 로 검토/재출력이 가능합니다.
- **오디오 싱크 플레이어는 아직 없습니다.** v1.x 후속 작업으로 검토합니다.
- **다중 작업 동시 변환 큐 는 아직 노출되지 않습니다.** GUI 는 한 번에 한 작업을 처리합니다.

## 8. 다국어

- **UI 문자열은 한국어 우선입니다.** 영어 번역은 v1.x 이후 검토 대상입니다.
- **STT 자체는 모델이 지원하는 모든 언어** 를 사용할 수 있습니다 (`--language` 옵션).

## 9. 알려진 자잘한 이슈

- 매우 짧은 (5초 미만) 오디오에서는 모델이 빈 결과를 돌려줄 수 있습니다. `--no-vad` 로 VAD 필터를 끄고 재시도해 보세요.
- 30분 이상 단일 세션 오디오에서 메모리 사용량이 모델 크기에 비례합니다. CPU `int8` 사용을 권장합니다.
- 원문 대조에서 동일 단어가 본문에 다수 등장하면 정렬 정확도가 떨어질 수 있습니다. 사용자 사전을 함께 사용하는 것이 좋습니다.

## 10. 원문 대조 / 자동 교정 적중률 (v1.0)

35분 분량 한국어 설교 + Markdown 원문에 대한 통합 회차에서 다음 패턴이 관찰됐습니다 (자세한 수치: [tests/integration/results.md](../tests/integration/results.md)).

- ~~**bible_refs 정규식 한계**: 원문이 `로마서 3: 21~22` 같은 콜론 표기를 쓸 경우 `parser._BIBLE_REF_RE` 가 `장` 키워드를 강제해 추출되지 않습니다.~~ — **2026-05-20 수정** (회차 #1 직후). `parser._BIBLE_REF_COLON_RE` 를 추가해 `로마서 3:21`, `로마서 3: 21~22`, `로마서 3:21-22` 형태를 모두 `로마서 3장 21-22절` 으로 정규화합니다. 단위 테스트: `tests/test_reference_parser.py::test_parse_extracts_bible_refs_colon_notation`.
- **한국어 자모(Jamo) Fuzzy 매칭 도입 (v1.0 완비)**:
  - 사전형 매칭의 한계(예: `이에수`, `그리시도`, `사다 바울`과 같은 자모 변형 오인식)를 해결하기 위해 **한글 자모 분해 및 초성 가중 평균 기반의 Hybrid Scorer를 활용한 Fuzzy 매칭 기능**이 v1.0에 정식 탑재되었습니다.
  - 이 기능은 Whisper 오인식 단어 감지율을 비약적으로 상승(통합 검증 결과 0건에서 292건으로 상승)시킵니다.
  - **False Positive (노이즈) 현상**: 임계값(`fuzzy_threshold`)을 기본값인 `0.70`으로 둘 때, 자모 구조가 단순한 3글자 성경 책 이름(예: 에스라, 에스더, 에스겔 등)이 본문의 짧은 발화("이시라" -> "에스라", "에서" -> "에스더")와 매칭되어 오추천을 발생시키는 현상이 관찰되었습니다.
  - **우회 및 제어**: 사용자는 GUI 설정 패널(더블 스핀박스) 또는 CLI 인자(`--fuzzy-threshold`)를 활용해 임계값을 `0.75`나 `0.80`으로 상향 설정하여 노이즈성 제안을 걸러낼 수 있습니다.
- **alignment_pairs 매치율**: 실제 발화와 원문 표현 차이가 크면 매치율이 낮을 수 있습니다. 회차 #1 에서는 641 세그먼트 중 11 건만 정렬됐습니다. 정렬되지 않은 세그먼트는 자동 교정 대상이 아니며, 편집기에서 직접 보완해야 합니다.
